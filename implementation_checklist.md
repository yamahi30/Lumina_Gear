# ContentHub AI 実装以外のやるべきことリスト

**作成日**: 2026年2月10日  
**対象**: sns_app_specification_v3.md に基づく

## 1. 外部サービスのアカウント作成・API Key取得

アプリが動くためには、以下のAPI Keyや認証情報が**実装前**に必要です。

| # | サービス | やること | 費用 | 所要時間目安 |
|---|---------|---------|------|-------------|
| 1 | **Google Cloud Console** | プロジェクト作成 → OAuth 2.0クライアントID取得 | 無料 | 30分 |
| 2 | ↑ | Google Drive API 有効化 | 無料 | 5分 |
| 3 | ↑ | Gmail API 有効化 | 無料 | 5分 |
| 4 | **Anthropic (Claude API)** | アカウント作成 → API Key取得 → クレジット入金 | 最低$5〜 | 15分 |
| 5 | **Google AI Studio (Gemini API)** | API Key取得 | 無料枠あり | 10分 |
| 6 | **X (Twitter) Developer** | Developer Portal申請 → Free tierのAPI Key取得 | 無料 | 1〜7日（審査あり） |
| 7 | **Meta (Threads API)** | Meta for Developersでアプリ作成 → アクセストークン取得 | 無料 | 1〜3日 |
| 8 | **Vercel** | アカウント作成（GitHub連携） | 無料 | 10分 |
| 9 | **Railway or Render** | アカウント作成 | 無料 | 10分 |

> **⚠️ 特に注意**: X Developer Portalの審査には**数日かかる場合がある**ので、最優先で申請してください。

## 2. 文体学習用の素材準備

AIに「あなたの文体」を教えるための素材が必要です。**6種類分**を事前に準備してください。

| # | 学習対象 | 準備するもの | 目安量 |
|---|---------|-------------|-------|
| 1 | **NOTE無料記事（アフィなし）** | 過去に書いた無料記事をコピペ用にテキスト保存 | 3〜5本分 |
| 2 | **NOTE無料記事（アフィあり）** | 過去のアフィリエイト記事をテキスト保存 | 2〜3本分 |
| 3 | **NOTEメンバーシップ記事** | メンバーシップ向けに書きたい文体の参考文章（自作 or 理想の他者記事） | 2〜3本分 |
| 4 | **NOTE有料記事** | 過去に書いた有料記事をテキスト保存 | 2〜3本分 |
| 5 | **X投稿** | 過去の反応が良かったX投稿を20〜30個コピペ | 20〜30投稿 |
| 6 | **Threads投稿** | 過去のThreads投稿を20〜30個コピペ | 20〜30投稿 |

> **💡 ポイント**: まだ投稿実績が少ない種類（メンバーシップ等）は、「こういう文体で書きたい」という理想の参考記事でもOK。

## 3. NOTE記事作成用のプロンプト設計

各NOTE作成機能の「設定ボタン」に登録するプロンプトを事前に考えておくと、Phase 2がスムーズです。

| # | 種類 | 考えるべきこと |
|---|------|---------------|
| 1 | 無料（アフィなし） | 記事構成ルール、トーン、禁止事項、文字数の指示 |
| 2 | 無料（アフィあり） | 上記 + アフィリエイト誘導の自然さの指示 |
| 3 | メンバーシップ | コミュニティ向けの親密感の出し方、パターン選択の指示 |
| 4 | 有料（お手頃/高額） | 価格帯別の構成ルール、画像挿入位置の指示方法 |

> **Phase 1の代替手段**: Claude.aiに4つのProjectを作成し、上記プロンプトをSystem Promptとして設定すれば、Phase 1中もNOTE記事作成は可能です。

## 4. コンテンツ戦略の整理

カレンダー生成AIに正しく指示するために、以下を**言語化して整理**しておく必要があります。

| # | 項目 | 具体的に決めること |
|---|------|------------------|
| 1 | **発信の柱（カテゴリ）** | 現状のカテゴリ一覧と配分比率の確認・調整（HSP共感21%、家庭DX23%等） |
| 2 | **各カテゴリの具体的なテーマ例** | AIが投稿アイデアを生成しやすいよう、カテゴリごとに5〜10個のテーマ例を用意 |
| 3 | **投稿時間帯** | X/Threadsの投稿時間帯の希望（現状: 7:30/12:30/21:00） |
| 4 | **ターゲット読者像** | 誰に向けた発信か（ペルソナ）を明文化 |
| 5 | **NG表現・避けたい話題** | AIに生成させたくない内容の明示 |
| 6 | **メンバーシップの方針** | 月額980円の価値をどう担保するか、コンテンツのバランス |

## 5. 競合調査の対象アカウント選定

設定画面に登録する競合アカウントを**事前にリストアップ**してください。

| プラットフォーム | 必要数 | 選定基準 |
|-----------------|--------|---------|
| **NOTE** | 3〜5アカウント | 同ジャンル（HSP/転職/家庭DX）で有料note販売している人 |
| **X** | 3〜5アカウント | 同ジャンルでフォロワー5,000〜50,000人程度の中規模アカウント |
| **Threads** | 2〜3アカウント | 同ジャンルで活発に発信している人 |

> **💡 選定のコツ**: 大手すぎる人より「少し先を行く先輩」的なアカウントが参考になります。

## 6. アフィリエイト案件の整理

NOTE記事（アフィあり）で使う商材情報を整理してください。

| # | 情報 | 例 |
|---|------|-----|
| 1 | カテゴリ名 | ITスクール、IT転職、動画サブスク、物販 |
| 2 | サービス名 | TechAcademy、レバテックキャリア等 |
| 3 | アフィリエイトURL | ASPから取得した専用URL |
| 4 | サービスの特徴 | 「無料カウンセリングあり」等 |
| 5 | 報酬単価（参考） | 成果判断のため把握しておく |

> **ASP登録がまだの場合**: A8.net、もしもアフィリエイト等への登録・案件提携も必要です。

## 7. 有料NOTE用のダウンロードコンテンツ準備

有料記事に添付する情報資産は**アプリでは作成しない**（仕様書記載の通り別途作成）ので、自分で用意が必要です。

| # | コンテンツ種類 | 準備すること |
|---|---------------|-------------|
| 1 | **Notionテンプレート** | テンプレートを作成 → 共有URLを取得 |
| 2 | **スプレッドシート（GAS付き等）** | シートを作成 → 共有URLを取得 |
| 3 | **ChatGPTプロンプト集** | テキストファイル or PDFで作成 → Google Driveに保存 |
| 4 | **その他ツール（VBA等）** | 作成 → ダウンロード可能な形式で保存 |

| # | やること | 備考 |

## 9. 運用ルールの策定

アプリ完成後にスムーズに運用するためのルール。

| # | 項目 | 決めること |
|---|------|----------|
| 1 | **投稿ルーティン** | 毎日いつアプリを開いて投稿を選ぶか |
| 2 | **カレンダー作成タイミング** | 毎月何日にカレンダーを生成するか |
| 3 | **投稿済みボタンの運用** | 投稿したら必ずボタンを押す習慣（学習精度に直結） |
| 4 | **メンバーシップ記事の公開曜日** | 毎週○曜日等、読者の期待感を作る |
| 5 | **分析レポートの確認タイミング** | 月曜朝にアカウント分析を確認する等 |
