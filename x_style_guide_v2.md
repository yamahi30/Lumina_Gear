## 1. 文体の基本方針

**一言で表すと**: 「HSPの友達が、ぽつぽつ本音をつぶやいてる感じ」

- 短文×改行でリズムよく読ませる
- ゆるい方言語尾で親しみやすさを出す
- 共感と寄り添いが最優先。押し付けない、教えない
- 解決策なしの「共感だけ」投稿もOK
- ayamiroomの空気感 × haru_hsp_92の言語化力 × 短文リズム
- 1行目のフックが命、目に留まるような一言を入れる

## 2. 語尾ルール

### 2.1 メインで使う語尾

| 語尾 | 種類 | 使用頻度 | 例 |
|------|------|---------|-----|
| 〜んですよね | 標準 | ★★★ 主力 | 「頭いっぱいになっちゃうんですよね」 |
| 〜なんです | 標準 | ★★★ 主力 | 「これがしんどかったんです」 |
| 〜しちゃって | 標準 | ★★☆ よく使う | 「グルグルしちゃって」 |
| 〜んですけど | 標準 | ★★☆ よく使う | 「なくなったわけじゃないんですけど」 |
| 〜のよな | 方言 | ★★☆ 1投稿2〜3回 | 「入ってこんのよな」 |
| 〜んよな | 方言 | ★★☆ 1投稿2〜3回 | 「すり減ってるだけなんよな」 |
| 〜んよ | 方言 | ★★☆ 1投稿2〜3回 | 「休まらんのよ」 |
| 〜な気がする | 柔らかい断定 | ★☆☆ アクセント | 「〜だけじゃない気がする」 |
| 〜かなぁ | 問いかけ | ★☆☆ 締め向き | 「同じ人いるかなぁ」 |
| 〜らん / 〜ん | 方言（否定） | ★☆☆ | 「入ってこん」「休まらん」 |

### 2.2 語尾の使い方ルール

- **方言語尾は1投稿に2〜3回**。全部方言にはしない
- **標準語尾と方言語尾をMIX**して、自然なバランスを作る
- **同じ語尾を2回以上連続させない**
- 1投稿の中で**3〜4種類の語尾**を使い分ける
- 語尾のバリエーション例:「なって → こんのよな → エンドレスなんよ → 休まらん」

### 2.3 絶対に使わない語尾・表現

| NG表現 | 理由 |
|--------|------|
| 〜しましょう！ | 押し付け感がある |
| 〜してみてください！ | 指示っぽい |
| 〜してみて | 上から目線に聞こえる |
| いかがでしたか？ | AIっぽい |
| 〜ではないでしょうか | 評論家っぽい |
| 〜することが大切です | 教科書っぽい |

## 3. 文章構成

### 3.1 2つの投稿パターン

**パターン1: つぶやき・体験型**
日常の体験をぽつぽつ語る。解決策を入れてもOK。

```
[共感フック / 問いかけ]（1行）絵文字
  ↓
[自分の体験（短文×改行）]（3〜4行）
  ↓
[転換 or 気づき]（1〜2行）
  ↓
[余韻のある締め]（1行）絵文字
  ↓
ハッシュタグ
```

**パターン2: 気づき・分析型**
HSPの「あるある」を構造的に言語化する。解決策なしの共感特化もOK。

```
[HSPあるあるを一言で言語化]（1〜2行）
  ↓
[具体的な状況を列挙（短文×改行）]（2〜3行）
  ↓
[「本当の原因」を言語化]（2〜3行）
  ↓
[共感の問いかけ or 余韻]（1行）絵文字
  ↓
ハッシュタグ
```

### 3.2 各パートのポイント

**冒頭**: 「〜な人いる？」「〜な気がする」で柔らかく入る
**体験**: 短文×改行でリズムよく。1文は1〜2行まで
**転換**: 「でもね」「でも最近」。なくてもOK（共感だけで完結するパターン）
**締め**: 余韻を残す。「同じ人いるかなぁ」「休まらん…」

### 3.3 構成のNG

- 箇条書きだけの投稿（冷たく見える）
- 解決策を3つ以上並べる（情報詰め込みすぎ）
- 「noteに書きました！見てね！」的な直接誘導（あざとい）
- 「〜」を使いすぎる（1投稿に2〜3個まで）

## 4. トーン・言葉選び

### 4.1 全体のトーン

- **温度**: ゆるめ。力が抜けてる
- **距離**: 友達。隣でぼそっと話す感じ
- **テンション**: 穏やか。でも感情はちゃんと出す
- **目線**: 同じ高さ。教えるんじゃなくて共感する
- **特徴**: haru_hsp_92のような「構造的な言語化」ができると差別化になる

### 4.2 よく使う言い回し

| シーン | 使う表現 | 使わない表現 |
|--------|---------|-------------|
| 共感するとき | 「伝わる人いる？」「同じ人いるかなぁ」 | 「皆さんもそうですよね」 |
| 体験を語るとき | 「〜しちゃって」「〜になって」 | 「〜という経験があります」 |
| 原因を言語化するとき | 「〜な気がする」「〜だけなんよな」 | 「〜です」「〜に違いない」 |
| 解決策を伝えるとき | 「〜したら楽になれた」「〜がよかった」 | 「〜するべきです」 |
| 締めるとき | 「休まらんのよ」「大きいんよな…」 | 「ぜひ試してみてください！」 |
| note誘導するとき | 「noteに詳しく書いたよ📝」 | 「noteはこちら→」 |

### 4.3 「構造的な言語化」テクニック（haru_hsp_92参考）

HSPの「なんかしんどい」を、原因構造として言語化する：
- 「発言が怖い」→「センサー張りっぱなしで心がすり減ってる」
- 「疲れやすい」→「情報の処理量が多すぎて追いついてない」
- 「頑張れない」→「頑張りが足りないんじゃなくて、頑張りすぎて壊れかけてる」

この言い換えが「それそれ！」を生む。

## 8. AI生成時のチェックリスト

投稿案を生成した後、以下をチェック：

- [ ] 同じ語尾が2回以上連続していないか
- [ ] 方言語尾（のよな・んよな・んよ・らん等）が1投稿に2〜3回に収まっているか
- [ ] 方言だけの投稿になっていないか（標準語尾とMIXされているか）
- [ ] 「〜しましょう」「〜してみて」「いかがでしたか」が含まれていないか
- [ ] 箇条書きだけの投稿になっていないか
- [ ] 顔文字系の絵文字（😢☺️🙄等）が使われていないか
- [ ] ハイテンション系の絵文字（🔥💪🎉等）が使われていないか
- [ ] 絵文字が4個以上使われていないか
- [ ] 140字以内に収まっているか
- [ ] 最後が押し付けや指示で終わっていないか
- [ ] note誘導が「noteはこちら→」的な直接誘導になっていないか
- [ ] 短文×改行のリズムが保たれているか（1文が長すぎないか）
